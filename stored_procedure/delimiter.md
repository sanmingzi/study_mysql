# delimiter

```m
在存储过程中，经常用到 delimiter。

在 mysql 中，分号的作用是告诉 mysql 解释器，该段命令已经结束了，可以执行了。默认情况下，delimiter 是分号。
但是有些时候，我们并不希望分号结束就开始执行，我们希望输入更多的 sql 语句。比如说存储过程。
存储过程中有多条语句，如果遇到分号就自动执行的话，那么存储过程就没法实现。
这种情况下，就需要事先把 delimiter 换成其他符号，常见的有 // 和 $$。

delimiter //
.....
//

这样，只有当出现"//"之后，mysql解释器才会执行这段语句，这样就能编写存储过程了。

修改sql语句结束符的定义
delimiter //     （结束符被定义为 “//”，遇到 // 执行sql语句）
delimiter ;      （结束符被定义为 “;”，遇到 ; 执行sql语句）
```
